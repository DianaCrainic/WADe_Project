Abstract

cryk is a cryptocurrency application which gives you personalized cryptocurrency information and news, analytics about the desired cryptocurrencies, to cut short your struggles. It gives vital details with great accuracy that otherwise take hours of calculation.

1 Introduction

1.1 Description

The cryk (cryptocurrency knowledge manager) application represents a modular Web system capable to express and manage the knowledge about existing cryptocurrencies and their (meta-)data. Each functionality will be implemented by a (micro-)service and all queries will be performed via a SPARQL endpoint. Exposed information will be available in multiple formats (e.g. HTML+RDFa and JSON-LD). Moreover, multiple visualizations of interest will be provided.

1.2 Motivation

Nowadays, cryptocurrency domain is attracting a lot of interest. Cryptocurrency is a decentralized digital currency stored in an online form that does not fall under governments and banks control, which was proposed in an attempt to replace the fiat currency.

Among thousands of cryptocurrencies, it can be overwhelming to decide which crypto to invest in. More often than not, people miss the important details and end up mismanaging their crypto investments. And that's especially true for serious investors with a widely spread-out portfolio.

2 Architecture & Design

2.1 Architecture

2.1.1 Description

The application architecture is based on microservices, with two microservices: Cryptocurrencies service and News service.

  1. Cryptocurrencies service is a service whose responsibility is to manage cryptocurrencies. It consists of an AWS Lambda function that performs various SPARQL queries (e.g. SELECT, INSERT, DELETE) on an Amazon Neptune database. The database contains RDF triples that model knowledge about existing cryptocurrencies and their (meta-)data. These RDF triples are loaded into the database through Amazon Neptune Bulk Loader (an external file uploaded to an Amazon S3 bucket will be loaded). Both the lambda function and the database runs within the same VPC (security reasons).
  2. News service is a service whose responsibility is to manage news about cryptocurrencies. It consists of an AWS Lambda function that performs various SPARQL queries (e.g. SELECT, INSERT, DELETE) on an Amazon Neptune database, like Cryptocurrencies service. The database contains RDF triples that model knowledge about crypto news. Both the lambda function and the database runs within the same VPC (security reasons).

The execution of the lambda functions associated with the microservices is triggered by calling the GraphQL API provided by AWS AppSync. Depending on the query or mutation that was called, the lambda function of the corresponding microservice will be executed. See GraphQL API: Schema for the schema structure of the GraphQL API.

A React client will call the GraphQL API and display the received data in a Single Page Application. An Amazon S3 bucket is used to store the static files built from the React application. The bucket is enabled for static hosting and is made public so the files can be accessed by the website endpoint. The website is served through an Amazon CloudFront distribution (a content delivery network (CDN) service built for high performance, security, and developer convenience).

The access to the application's features is controlled using Amazon Cognito. See Application-Design: Users & Use Cases to learn more about the user types of the application as well as the features they can access.

2.2.2 Users & Use Cases

There are 2 types of users: the unauthenticated user and the admin (which it is an authenticated user with more permissions).
The final user, which in this case is an unauthenticated user can:

  • visualize the list of the existing cryptocurrencies, action which includes the following:
    • compare 2 desired cryptocurrencies based on various criteria
    • view the volume of the desired cryptocurrency
    • view the relevant news about the desired cryptocurrency
  • export the data in the desired format
The admin can perform the following actions, in addition to those that the unauthenticated user can do:
  • create a new desired cryptocurrency
  • update the desired cryptocurrency
  • delete the desired cryptocurrency
  • create a piece of news about a cryptocurrency
  • update a piece of news about a cryptocurrency
  • delete a piece of news about a cryptocurrency

2.2.3 I/O Data Formats

For the Cryptocurrency Service, a file in the format n-triples (with extension .nt) is loaded from an Amazon S3 bucket in Amazon Neptune. The lambdas query Neptune using SPARQL and receive the result as RDF and/or JSON. The front-end sends JSON with a GraphQL query to AWS AppSync. AppSync then sends the query further, to the corresponding lambda of the operation. The lambda sends back to the client JSON(-LD). The webpage presents to the user data in JSON-LD, RDFa formats and allows them to export it as a .csv file.

3 Data Sources

3.1 Internal Data Structures/Models

Application data can be accessed and manipulated through a GraphQL API. Each query or mutation call from the API will be directed to the corresponding microservice based on the data required for the operation, therefore 2 models/entities have been defined: Cryptocurrency, which describes a cryptocurrency, with all its (meta-)data, and CryptoNews, which depicts a news article regarding a cryptocurrency. These 2 models are based on appropiate RDF entities, namely DOACC's Cryptocurrency and Schema.org's NewsArticle, respectively.

3.2 External Data Sources

Information regarding cryptocurrencies is procured from DOACC (Description Of A CryptoCurrency), a catalogue of published facts about altcoins (cryptocurrencies other than Bitcoin), like name, trading symbol, total coins, block target time, hashing algorithm, date launched, URL of the source code repository, etc. It is published using semantic web technology, which takes advantage of the attendant power and flexibility of representation in order to improve ease of re-use.

4 Linked Data Principles

The Linked Data principles refer to a set of best practices for publishing structured data on the Web. The rules, which are as follows:

  1. Use URIs as names for things.
  2. Use HTTP URIs so that people can look up those names.
  3. When someone looks up a URI, provide useful information, using the standards (RDF, SPARQL).
  4. Include links to other URIs. so that they can discover more things.
are meant to implement the Semantic Web, which revolves around creating links, so that a person or machine can explore the web of data.

To evaluate the data openness and essentially whether these principles are followed closely or not, a star rating has been designed, known as 5 ★ Open Data. It is comprised of 5 quality levels, rewarding from 1 to 5 stars based on how open, public, easy to use and rich in context is the data offered.

Our team strives to offer 5 ★ Open Data, rich in context and easy to be used by people and machines alike. In order to do so, we provide (meta-)data on cryptocurrencies and news related to them and make the information available on the Web with an open license (★). Moreover, the facts are offered as machine-readable structured data (★★), in a non-proprietary format (★★★) which uses the RDF standard to identify things (★★★★) that link to other datasets, according to the linked data initiative (★★★★★).

7 Technical aspects concerning the implemented API

The GraphQL API implemented within the application is provided through AWS AppSync, a fully managed GraphQL API layer developed by Amazon Web Services. AppSync allows developers to build GraphQL APIs without much of the usual work; it handles the parsing and resolution of requests as well as connecting to other AWS services like AWS Lambda, NoSQL and SQL data stores, and HTTP APIs to gather backend data for the API.

As far as the current project is concerned, any call to the API determines the invocation of a lambda function (either the function associated with the cryptocurrencies service, or the function associated with the news service) since each query or mutation is connected to a lambda function data source.

The mapping between the lambda function that is called and the operation is the following:

  • cryptocurrencies-service-lamda
    • Queries
      • cryptocurrency - used to retrieve details about a specific cryptocurrency (e.g. symbol, description, total coins)
      • cryptocurrencies - used to retrieve details about multiple cryptocurrencies
      • cryptocurrenciesInfo - used to additional information about cryptocurrencies (e.g. how many they are)
    • Mutations
      • createCryptocurrency - used to create a new cryptocurrency
      • updateCryptocurrency - used to update an existing cryptocurrency
      • removeCryptocurrency - used to remove a cryptocurrency
  • news-service-lambda
    • Queries
      • cryptoNews - used to retrieve news associated with a cryptocurrency
      • cryptoNewsInfo - used to retrieve additional information about news associated with a cryptocurrency (e.g. how many they are)
    • Mutations
      • createCryptoNews - used to create a new piece of news related to a cryptocurrency
      • updateCryptoNews - used to update an existing piece of news related to a cryptocurrency
      • removeCryptoNews - used to remove a piece of news related to a cryptocurrency

This GraphQL API is called by the React application through Apollo Client library. Apollo Client is a comprehensive state management library for JavaScript that enables you to manage both local and remote data with GraphQL. Use it to fetch, cache, and modify application data, all while automatically updating your UI.

The React application takes full advantage of the features brought by this library. For example, the cryptocurrencies displayed on an accessed page are cached. Therefore, upon subsequent access to the same page, the cryptocurrencies that will be displayed will not be obtained through a call to the API, but will be retrieved from the cache. Another example which is worth mentioning: the library will automatically call the API to get the updated state of the page when a cryptocurrency is deleted from the current page.
