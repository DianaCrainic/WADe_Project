Abstract

cryk is a crypto application which gives you personalized cryptocurrency information and news, analytics about the desired cryptocurrencies, to cut short your struggles. It gives vital details with great accuracy that otherwise take hours of calculation.

1 Introduction

1.1 Description

The cryk (cryptocurrency knowledge manager) application represents a modular Web system capable to express and manage the knowledge about existing cryptocurrencies and their (meta-)data. Each functionality will be implemented by a (micro-)service and all queries will be performed via a SPARQL endpoint. Exposed information will be available in multiple formats (e.g. HTML+RDFa and JSON-LD). Moreover, multiple visualizations of interest will be provided.

1.2 Motivation

Nowadays, cryptocurrency domain is attracting a lot of interest. Cryptocurrency is a decentralized digital currency stored in an online form that does not fall under governments and banks control, which was proposed in an attempt to replace the fiat currency.

Among thousands of cryptocurrencies, it can be overwhelming to decide which crypto to invest in. More often than not, people miss the important details and end up mismanaging their crypto investments. And that's especially true for serious investors with a widely spread-out portfolio.

2 Architecture & Design

2.1 Architecture

2.1.1 Description

The application architecture is based on microservices, with two microservices: Cryptocurrencies service and News service.

  1. Cryptocurrencies service is a service whose responsibility is to manage cryptocurrencies. It consists of an AWS Lambda function that performs various SPARQL queries (e.g. SELECT, INSERT, DELETE) on an Amazon Neptune database. The database contains RDF triples that model knowledge about existing cryptocurrencies and their (meta-)data. These RDF triples are loaded into the database through Amazon Neptune Bulk Loader (an external file uploaded to an Amazon S3 bucket will be loaded). Both the lambda function and the database runs within the same VPC (security reasons).
  2. News service is a service whose responsibility is to manage news about cryptocurrencies. It consists of an AWS Lambda function that performs various SPARQL queries (e.g. SELECT, INSERT, DELETE) on an Amazon Neptune database, like Cryptocurrencies service. The database contains RDF triples that model knowledge about crypto news. Both the lambda function and the database runs within the same VPC (security reasons).

The execution of the lambda functions associated with the microservices is triggered by calling the GraphQL API provided by AWS AppSync. Depending on the query or mutation that was called, the lambda function of the corresponding microservice will be executed. See GraphQL API: Schema for the schema structure of the GraphQL API.

A React client will call the GraphQL API and display the received data in a Single Page Application. An Amazon S3 bucket is used to store the static files built from the React application. The bucket is enabled for static hosting and is made public so the files can be accessed by the website endpoint. The website is served through an Amazon CloudFront distribution (a content delivery network (CDN) service built for high performance, security, and developer convenience).

The access to the application's features is controlled using Amazon Cognito. See Application-Design: Users & Use Cases to learn more about the user types of the application as well as the features they can access.

2.2.2 Users & Use Cases

There are 2 types of users: the unauthenticated user and the admin (which it is an authenticated user with more permissions).
The final user, which in this case is an unauthenticated user can:

  • visualize the list of the existing cryptocurrencies, action which includes the following:
    • compare 2 desired cryptocurrencies based on various criteria
    • view the volume of the desired cryptocurrency
    • view the relevant news about the desired cryptocurrency
  • export the data in the desired format
The admin can perform the following actions, in addition to those that the unauthenticated user can do:
  • create a new desired cryptocurrency
  • update the desired cryptocurrency
  • delete the desired cryptocurrency
  • create a piece of news about a cryptocurrency
  • update a piece of news about a cryptocurrency
  • delete a piece of news about a cryptocurrency

5 Used Technologies

5.1 Node.js

Node.js is an asynchronous event-driven JavaScript runtime, which is designed to build scalable network applications. The microservices from the current application are developed using Node.js.

5.2 React

React is a declarative, efficient, and flexible JavaScript library for building user interfaces. React can be used as a base in the development of single-page, mobile, or server-rendered applications with frameworks like Next.js.

In this context, a React client will call the GraphQL API and display the received data in a Single Page Application (the website).

5.3 AWS services

5.3.1 Amazon Neptune

Amazon Neptune is a fully managed database service built for the cloud that makes it easier to build and run graph applications. Neptune provides built-in security, continuous backups, serverless compute, and integrations with other AWS services. The database contains RDF triples that model knowledge about existing cryptocurrencies and their (meta-)data and also about crypto news.

5.3.2 AWS Lambda

AWS Lambda is a serverless, event-driven compute service where the code is run virtually for any type of application or backend service without provisioning or managing servers. It is responsible for managing data from Amazon Neptune database using various SPARQL queries (e.g. SELECT, INSERT, DELETE).

5.3.3 AWS AppSync

AWS AppSync creates serverless GraphQL and Pub/Sub APIs that simplify application development through a single endpoint to securely query, update, or publish data. The GraphQL API provided by AWS AppSync is called when the execution of the lambda functions associated with the microservices is triggered Depending on the query or mutation that was called, the lambda function of the corresponding microservice will be executed.

5.3.4 Amazon S3

Amazon Simple Storage Service (Amazon S3) is an object storage service offering industry-leading scalability, data availability, security, and performance. Customers of all sizes and industries can store and protect any amount of data for virtually any use case, such as data lakes, cloud-native applications, and mobile apps. With cost-effective storage classes and easy-to-use management features, you can optimize costs, organize data, and configure fine-tuned access controls to meet specific business, organizational, and compliance requirements.

In this context, Amazon S3 bucket is used to store the static files built from the React application. The bucket is enabled for static hosting and is made public so the files can be accessed by the website endpoint. Besides that, an Amazon S3 bucket is used to upload RDF triples containing knowledge about cryptocurrencies to Amazon Neptune. The RDF file is stored in the s3 bucket and loading is done through Amazon Neptune Bulk Loader.

5.3.5 Amazon Cloudfront

Amazon CloudFront is a content delivery network (CDN) service built for high performance, security, and developer convenience. When a user requests content that is served with CloudFront, the request is routed to the edge location that provides the lowest latency (time delay), so that content is delivered with the best possible performance.

5.3.6 Amazon Cognito

Amazon Cognito provides an identity store that provides user sign-up and sign-in features and control access to the web applications. It scales to millions of users, supports social and enterprise identity federation, and offers advanced security features to protect the consumers and business. Built on open identity standards, Amazon Cognito supports various compliance regulations and integrates with frontend and backend development resources.
